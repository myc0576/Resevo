# Experiment Review

用途：在实验或算法任务结束后，对输入、流程、输出和验证证据做结构化复盘。

## Trigger

当任务涉及实验数据、图像处理、标定、重建、可视化、指标或论文证据时使用。

## Procedure

1. 复制 `templates\experiment_review_template.md`。
2. 记录 task goal、input、process、output。
3. 记录验证证据和未验证边界。
4. 判断是否需要 knowledge capture、prompt capture、research asset harvest 或 decision record。

## Boundary

Review 不等于结论证明。没有证据的推断必须标记为 `hypothesis` 或 `pending validation`。
